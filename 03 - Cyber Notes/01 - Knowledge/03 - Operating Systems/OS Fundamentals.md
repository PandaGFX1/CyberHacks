- Operating System (OS) - Controls most functions of a computer 
    

User/Application 

- User – The person using a computer/device. User interaction is provided through OS interfaces. Most common interfaces include a GUI or CLI. 
    

- Application – Executable code designed to carry out specific tasks on the computer. Can be referred to as programs or software 
    

System Software 

- System Software – Another name for Operating System. Consists of: 
    

- OS Interface – What you see; Allows a user to interface with applications, system, software, and the hardware 
    

- OS Kernel – The core component of an OS, and manages the interaction between user, applications, and hardware 
    

- Hardware – The mechanical, magnetic, electronic, and electrical components of a computer 
    

OS Fundamentals 

Process Management 

- Process – Executing instance of an application. OS creates and manages a processes resources. When the process is closed the OS then frees up the previous allocated resources  
    

Device Management 

- Device – ANY hardware component or peripheral attacked to a computer 
    

- Device Driver – Computer program that operates or controls a device attached to a computer. Driver is used by the OS as an interpreter 
    

File System Management 

- File System – Complete director structure, which includes a root directory and any subdirectories and files beneath it. Defines how data is named, stores, organized, and accessed 
    

- Physical Disk – Physical hard drive that is the primary storage hardware/component within a computing device. Used to store, retrieve, and organize data. 
    

- Disk Volume – A volume is the part of the disk that you interact with as a user. File systems are confined to a single logical volume 
    

- Directories – A container used to organize files and data for use by an OS 
    

- Files/Data - A singular named item used to store information/data 
    

OS Categories  

- Single User 
    
    - Single-Tasking – It is an operating system in which the user can manage one thing at a time effectively 
        
    - Multi-Tasking – Permits several users to utilize the programs that are concurrently running on a single network server 
        

- Multiuser – An operating system which allows MULTIPLE users to access the single system with one operating system in it. It is generally used on large mainframe computers. 
    

- Client – Operates in a networked environment allowing a user to connect to another computer (server) to retrieve information. 
    

- Server – An OS specifically designed to run on servers, security, to other computers across a network 
    

- Cluster – Two or more servers that are grouped together, operating as one balancing loa of work being conducted with load balancing 
    

- Distributed – Interconnected computers communicating with eachother using a shared communication network 
    

DOD IA Measures 

- DOD IA – Mandatory measures that protect alongside and help to defend computer systems 
    

- Integrity – Assurance that ALL information systems are protected and not tampered with 
    

- Availability – Process of ensuring users have access to data and services  
    

- Authentication – Properly verifying a user's credentials. This involves strong passwords, two-factor authentication, bio-metric devices, and other similar methods 
    

- Confidentiality – Only those with authorization may view certain data (Confidential) 
    

- Non-Repudiation – Someone with access to your organization's information system cannot deny a task they have conducted 
    

Privileged Accounts 

- Administrator – The PRIVILEGED account in Windows 
    

- Root – The PRIVILEGED account in Linux 
    

Non-Privileged Accounts 

- Principle of Least Privilege – Non-privileged user accounts are only granted access to what is authorized. Works by allowing only enough access to perform the required job. - Reduces the risk of attackers gaining access to sensitive information or compromising privileged accounts 
    

Access Control 

- Discretionary Access Control – Only one that uses the Access Control List (ACL) 
    
    - Access Control List (ACL) -  Details account permissions to a specific resource as determined by permissions set by the owner of that resource. 
        

- Mandatory Access Control (MAC) - Set by an administrator using a policy tool. Users cannot change the policies or  
    

- Role-Based Access Control – Access decisions are based on each account's role. (EX: Those with a student role may have less access to admin information than an instructor would) 
    

Characteristics Of Operating Systems 

- Memory Management – Keeps track of the primary memory and allocates the memory when a process or program requests it (EX: what part of it is in use by whom, what part of it is not in use, etc.)  
    

- Processor Management – Allocates the processor (CPU) to a process and deallocates the processor when it is no longer required 
    

- Device Management – Keeps track of all the devices. This is also called I/O controller that decides which process gets the device, when, and for how much time 
    

- File Management – Allocates and de-allocates the resources and decides who gets the resources 
    

- Security – Prevents unauthorized access to programs and data by means of passwords and other similar techniques 
    

- Job Accounting – Keeps track of time and resources used by various jobs and/or users 
    

- Control Over System Performance – Records delays between the request for a service and from the system 
    

- Interaction with the Operators – Interaction may take place via the console of the computer in the form of instructions. The OS acknowledges the same, does the corresponding action, and informs the operation by a display screen 
    

- Error-Detecting Aids – Production of dumps, traces, error messages, and other debugging and error-detecting methods. 
    

- Coordination Between Other Software and Users – Coordination and assignment of compilers, interpreters, assemblers, and other software to the various users of the computer systems
