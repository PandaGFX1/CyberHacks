Tags: #CompArch #Terminology 

Thread – Basic unit to which the operating system allocates processor time. A process can have multiple threads. - Threads share memory and resources when a process starts. 

Multithreaded Processes – Intelligent scheduling of instructions from two FDE data paths. Whichever thread is ready first goes first. 

Process – A program in execution. Must progress in a sequential fashion. You write a program and when you execute that program it becomes a process. (EX: Background Tasks such as Spell Checker, Google Chrome, Firefox) 

Programs – Comprised of multiple processes running. Group of ordered operations. Longer lifespan than a process. Contains MANY resources. Programs pull their data from memory. 

Multicore Processing – Consists of two or more processing units on a single chip. Allows instructions to execute simultaneously.  

Multi-Processing – Processing of data using two or more CPUs (or CPU Cores). Requires you to have load balancing (divides memory) and scheduling (control whose turn it is) 

Parallel Computing – Tasks are broken down into smaller tasks that can all be done at the same time. Then broken down into even smaller instructions to execute simultaneously.  

Flynn's Taxonomy – 4 classifications of computers and how streams of information work. Based on the notion that there are 2 streams of information (Instructions and Data) 

Graphics Processing Unit (GPU) - Processor that is programmable and used to display images on your computer screen. Contains its own processor and RAM.
