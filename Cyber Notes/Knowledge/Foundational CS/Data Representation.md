- Sizing
	- Bit - 1 Binary Digit 
	- Nibble - 4 Bits (EX: 1101) 
	- Byte - 8 Bits (EX: 1101 1110) 
	- Word - 16 Bits or 2 Bytes (EX: 1101 1111 0000 0101) 
	- Double Word - 32 Bits or 4 Bytes or 2 Words 
	- Quad Word - 64 Bits or 8 Bytes or 4 Words or 2 Double Words 
	- **Everything essentially just doubles. Base 2 or everything to the 2nd power
	- Position Number - A system for establishing how numeric values are represented. (EX: Binary) 
	- Base - Simplest structure of a number system (EX: Base 10) 
	- Most Significant Bit - The "bit" or number to the leftmost side 
	- Least Significant Bit - The "bit" or number to the rightmost side
- Number Systems:
	- Binary - Base 2; Computer's language; Represented by 1's and 0's. 
	- Octal - Base 8; Uses 0-7; Just do 8^0, 8^1, and so on. 
	- Denary - Base 10; Also known as decimal. It's what we use every day. 1-10. 
	- Hexadecimal - Known as Base 16; Helps to simplify binary/big numbers. 16^0, 16^1, and so on; Uses letters and numbers. (0-9 + A-F)
- Signed Arithmetic/Signed Integers:
	- The most significant value represents the sign (+/-). If there is a 1 then the number is a -. Positive would be 0. If there is a signed integer, then instead of 8 bits of data, you would have 7. 
	- Absolute values do exist as well.
- Floating Point Arithmetic
	- Floating Point Arithmetic  
	- Floating Point Number - Any real number with a decimal 
	- Mantissa - Part of the number stored after the decimal point 
	- You have the sign bit (1 value), the exponent value, and then the mantissa. 
	- Used for very small/large numbers that require fast processing. 
	- Uses IEE 754 Floating Point Standard, which has 2 main types of precision (32 bit and 64 bit)
